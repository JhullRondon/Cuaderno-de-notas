Algoritmos

En informática, un algoritmo es una secuencia de instrucciones secuenciales, gracias al cual pueden llevarse a cabo ciertos procesos y darse respuesta a determinadas necesidades o decisiones. Se trata de conjuntos ordenados y finitos de pasos, que nos permiten resolver un problema o tomar una decisión. Los algoritmos no tienen que ver con los lenguajes de programación, dado que un mismo algoritmo o diagrama de flujo puede representarse en diversos lenguajes de programación, es decir, se trata de un ordenamiento previo a la programación. Visto así, un programa no es otra cosa que una serie compleja de algoritmos ordenados y codificados mediante un lenguaje de programación para su posterior ejecución en un computador. Los algoritmos también son frecuentes en la matemática y la lógica, y son la base de la fabricación de manuales de usuario, folletos de instrucciones, etc. Su nombre proviene del latín algoritmus y éste apellido del matemático persa Al-Juarismi. Uno de los algoritmos más conocidos de la matemática es el atribuido a Euclides, para obtener el máximo común divisor de dos enteros positivos, o el llamado “método de Gauss” para resolver sistemas de ecuaciones lineales.

Metodología de solución de un algoritmo

  • Definición del problema: delimitar el problema que vamos a resolver
  • Análisis: identificar todos los elementos que se relacionan en nuestro problema
  • Diseño del algoritmo: darle forma al algoritmo
  • Verificación o pruebas: someter al pruebas al algoritmo y comprobar que funcione correctamente

Metodología para extraer datos de un algoritmo

  • Entrada: que se necesita para realizar los pasos
  • Salida: que se obtiene al final del algoritmo
  • Tipo de datos: saber qué tipo de dato manejara el algoritmo como: números, textos, otro…

Objetivos del análisis de algoritmos

  • Resolver el problema antes de empezar la primera línea de código
  • Buscar las posibles soluciones y seleccionar la mas eficiente
  • Reducir tiempo de verificación y corrección

Como comparar algoritmos

La mejor forma de comparar algoritmos es mediante el ritmo de crecimiento y para realzar este análisis siempre hay que tomar en cuenta el término de mayor peso

Estructuras de control

Nos permiten definir el flujo de nuestro algoritmo, ejecutar acciones en la parte del proceso correcta. Se dividen en 3 grupos:

Estructuras secuenciales: consiste en ejecutar las instrucciones de principio a fin, una a una sin omitir pasos o desviarse del proceso definido.

Estructuras selectivas

Nos van a permitir que nuestro algoritmo o programa tome decisiones en el caso de ejecutar un pedazo de código o un segmento de código o no con base en una condición específica que debemos verificar si se cumple, y si se cumple ejecutamos la codicion y si no se va colocando lo siguiente de nuestro código.

  • Si, if: Aqui vamos a tener una condición y en caso de que la condición se cumpla vamos a ejecutar un pedazo de código. En caso contrario saltamos esa parte del código.
  • Si-sino, if-else: nos permite una condición y bifurcar el camino para 2 rutas distintas, es decir, si la condición se cumple ejecutamos una acción y se la condición no se cumple ejecutamos otra acciones diferente.
  • Si, sino entonces sino, if-else if- else: aquí vamos a poder verificar la primera condición en caso de que la primera no se cumpla, podemos verificar una segunda condición y en caso de que no se cumpla podemos tener n condiciones. Si este número n que hayamos definido no se cumplen al final se ejecutara un else

Estructuras de selección multiple:
Estas estructuras nos van a permitir concadenar tomar decisiones y al final tener una decisión estándar, en el caso de que no se haya entrado en ninguna de las condiciones.

Estructuras repetitivas

Estás nos van a permitir crear bucles donde nuestros programas va a estar repitiéndose hasta que una condición les indique lo contrario o mientras una condición les indique que hagamos eso.

  • Mientras, while. Mientras una condición se cumpla, nosotros vamos a ejecutar todo el código que está dentro de este ciclo y vamos a repetirlo.
  • Haz mientras, do while. Esta instrucción nos permite entrar por lo menos una vez a ejecutar el código que está dentro de esta instrucción y si la condición se cumple , entonces repetimos, pero si la condición no se cumple salimos del código.
  • Para, for. Se utiliza cuando necesitamos repetir una serie de veces definida un código.La diferencia de un “mientras” y un “haz mientras” es que el “haz mientras” se va a ejecutar por lo menos una vez y el “mientras” no necesariamente se va a ejecutar, si la condición no se está cumpliendo no va a entrar ni una vez a ese código y lo saltaría.
